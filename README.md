# Telecom Customer Churn Analysis Using SQL

## Project Overview

This project analyzes customer churn behavior in a telecommunications company using SQL Server and SQL Server Management Studio (SSMS).

The primary objective of this project is to identify churn patterns, evaluate revenue impact, analyze customer behavior, segment customers based on risk and value, and generate business recommendations that support customer retention strategies.

---

## Dataset

**IBM Telco Customer Churn Dataset**

The dataset contains customer demographic information, account details, subscribed services, billing information, contract details, and churn status.

Key fields analyzed include:

* CustomerID
* Gender
* SeniorCitizen
* Partner
* Dependents
* Tenure
* PhoneService
* InternetService
* OnlineSecurity
* TechSupport
* Contract
* PaymentMethod
* MonthlyCharges
* TotalCharges
* Churn

---

## Business Objective

The goals of this project are:

* Measure overall customer churn.
* Identify customer segments with higher churn risk.
* Analyze customer retention patterns.
* Evaluate revenue impact caused by churn.
* Compare churn across contracts, services, and customer groups.
* Generate data-driven business recommendations.

## Key Findings

* Customer churn varies significantly across customer segments.
* Contract type plays an important role in customer retention.
* Payment method influences churn behavior.
* Customer tenure is a strong indicator of retention likelihood.
* Service adoption impacts customer loyalty.
* Customer churn creates measurable revenue loss.
* Customer segmentation helps identify high-risk customer groups.
* Revenue analysis helps prioritize retention efforts.

---

## Business Recommendations

* Prioritize retention efforts for high-risk customer groups.
* Improve onboarding strategies for new customers.
* Promote value-added services such as Tech Support and Online Security.
* Monitor churn trends by contract type and payment method.
* Focus retention campaigns on high-value customers.
* Develop churn monitoring dashboards for continuous tracking.

---

## Conclusion

This project demonstrates the use of SQL for customer churn analytics, revenue analysis, customer segmentation, KPI development, business reporting, and data-driven decision making.

The analysis highlights how customer demographics, services, contract structures, and revenue patterns can be used to better understand churn behavior and improve customer retention strategies.
